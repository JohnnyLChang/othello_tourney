File specification

Othello_Core.py is the template file that all students will have access to. You can access it from you code using import Othello_Core. It lays out the basic constants and a class that all student submissions must inherit from, and methods that must be overwritten.

Students must create a file called strategy.py with a class called Strategy (optionally inheriting from Othello_Core.OthelloCore) and containing a method called best_strategy taking the arguments board, player, best_move, still_running in that order. best_move and still_running are multiprocessing.Value objects, and can be changed using best_move.value=34 for example. still_running is a flag turned to 0 when the code is about to be killed.

As a quick example, here is the most basic form of AI, random choice: 

import Othello_Core as core
import random

class Strategy(core.OthelloCore):
    # implement all the required methods on your own
    def best_strategy(self, board, player, best_move, running):
        best_move.value = random.choice(self.legal_moves(board, player))

Other Info

If you code fails to return a move, returns an invalid move, or errors for whatever reason, it will be treated as a forfeit. As such, make sure you code works on your computer before using this website to test against other ais. Just as a reminder, if your code is caught cheating during the tournament in any way, you will be caught and given an integrity violation, so just don't do it.
